# 内在宇宙 · 完整蓝图

> Product Blueprint v1.0  
> 本文件定义《内在宇宙》完整产品应当成为怎样的世界、用户如何在其中行动，以及各系统之间如何协同。  
> 优先级低于《核心哲学理念》，高于具体界面、视觉资产与开发任务。

## 1. 产品定义

《内在宇宙》（Inner Universe）是一个以荣格分析心理学、主动想象与心理外化为灵感的沉浸式心理探索空间。

它不是心理测评、情绪日记、效率工具、聊天机器人或治疗替代品。它将人的情绪、情结、阴影、防御、关系模式与尚未被意识理解的内容，转化为一个可被观看、停留、命名、探索和逐步显影的个人内在世界。

产品最终应让用户产生一种体验：

> 我不是在管理我的情绪。  
> 我正在与一个比意识更大的自己相处。

## 2. 核心价值

产品不替用户提供“正确的人格解释”，而是提供四种能力：

- **外化**：让难以言说的心理经验变成可观看的对象。
- **距离**：让用户从“被情绪吞没”转向“我正在观察一个情绪对象”。
- **关系化**：让用户看见对象之间的牵动、靠近、回避和重复模式。
- **整合**：让用户不以消灭问题为目标，而以理解、命名、容纳和重新组织为目标。

## 3. 用户与使用场景

核心用户是有内省倾向、对心理学或自我探索感兴趣、对传统聊天型 AI 或标准化测评感到疏离的人。

典型场景包括：睡前短暂回看今天的内在状态；情绪波动时进入一个对象并停留；在长期关系、职业或自我探索中观察重复模式；隔一段时间拉远视角，查看自己的宇宙如何发生变化。

产品不追求高频打卡。一次短暂、安静且真实的停留，比连续使用天数更重要。

## 4. 世界模型

整个产品只有一个世界：用户的内在宇宙。

它不是由多个独立页面拼接而成，而是一个连续的意识空间。宏观、中观、微观和镜像层只是用户与同一世界之间的不同心理距离。

```text
宏观宇宙 Macro
    ↓ 缩放 / 进入
中观区域 Region
    ↓ 聚焦 / 凝视
微观对象 Object
    ↓ 停留 / 显影
意识层 Dialogue
```

系统的核心交互不是“打开页面”，而是“改变意识距离”。

## 5. 四层体验结构

### 5.1 宏观宇宙：看见整体

宏观层是产品首页。用户看到的是局部可见的心理宇宙，而不是一张完整地图。

此层承担的任务是：建立 Self 的稳定中心、呈现当前可见的心理对象、保留未知区域、允许用户拖拽与缩放，并让用户隐约感受到对象之间存在关系。

用户在此层的主要感受应是：

> 我的内在不是混乱的，它有自己的空间、距离和引力。

### 5.2 中观区域：进入关系

中观层不是“对象详情页”，而是一个局部心理区域。

用户可能进入亲密关系、创造力、工作压力、家庭记忆或某个未命名区域。此时宏观星图的关系结构开始变成具有空间感的心理地形：裂缝、沉积物、晶体、纤维、阴影雾、缓慢流动的记忆带。

用户在此层不只是看见一个对象，而是开始感受对象所处的关系环境。

### 5.3 微观凝视：与对象相处

微观层是《内在宇宙》最独特的体验。

用户进入一个具体心理对象后，绝大多数 UI 消失。对象成为场景中心。用户不被要求立即输入、解释或完成任务，而是被邀请停留、旋转、靠近和凝视。

对象并不会一次性完整展示。用户停留越久、回访越多、对话越深入，内部的裂纹、记忆纤维、粒子、象征图案和微弱语言线索才逐步显影。

用户感受应是：

> 我没有在分析它。  
> 我正在陪伴它显现。

### 5.4 意识层：镜像与主动想象

意识层由“镜像”承载。镜像不是人格化 AI，也不是聊天机器人。

它是一层液态光膜或意识界面，存在于用户与对象之间。用户的语言以冷静、清晰的意识痕迹进入空间；镜像的回应以温和、开放、非权威的方式浮现。

镜像只负责三个动作：照亮、停留、命名。它不诊断、不归因、不替用户下结论。

## 6. 心理结构映射

以下映射用于保持世界的一致性，不用于给用户贴标签。

| 心理结构 | 宇宙/视觉对应 | 产品含义 |
|---|---|---|
| Self / 自性 | 安静的暖金种子或稳定内核 | 整体人格的中心，不是太阳或权威 |
| Ego / 意识自我 | 用户的观察位置与镜头 | 看见、命名、选择靠近或离开 |
| 情结 / Complex | 独立心理对象、星体或结构 | 带有情绪能量和个人历史的心理单元 |
| 模式 / Pattern | 关系场、局部星座、共同节律 | 多个对象长期重复形成的关联 |
| 原型 / Archetype | 大尺度结构、旋臂或潜流 | 深层人格倾向与集体心理结构 |
| Shadow / 阴影 | 暗紫雾、低对比无边界结构 | 未被承认但持续产生影响的部分 |
| Trauma / 创伤 | 裂纹琥珀、空间异常、局部黑区 | 高能量压缩、难以直接凝视的经验 |
| Defense / 防御 | 雾、遮蔽、折射和模糊边界 | 曾经保护用户的心理策略 |
| Memory / 记忆 | 冷白颗粒、纤维、封存层 | 被保存、被压缩或等待显影的经验 |
| Insight / 洞见 | 微青光、局部清晰化与新连接 | 意识开始形成新的理解 |

## 7. 心理对象生态

对象不是统一节点，也不是用户手动创建的标签容器。每个对象都必须具有独特形态、生命节律、关系方式和显影路径。

第一批基础对象可包括：

- **创伤**：裂纹半透明琥珀，内部有冻结粒子和被压缩的光。
- **创造力**：持续生长的透明枝晶、纤维与微弱冷白尖端。
- **亲密关系**：两个彼此靠近、缠绕、分离的半透明结构。
- **逃避**：碎裂、漂浮、边界模糊且逐渐远离中心的结构。
- **阴影**：无法完全看清、在紫雾中不断改变边界的低对比存在。

后续对象可由用户命名，但系统不应默认给出确定性心理结论。

## 8. 探索与发现机制

### 8.1 发现而非生成

对象不是被系统“创建”，而是被意识逐步发现。

系统不说“已检测到某种问题”，而是通过微光、粒子沉积、雾气松动、轮廓出现等方式，暗示一个区域开始被照亮。

### 8.2 停留而非解锁

重要内容不通过任务、按钮或成就系统获得。

用户的停留、回访、旋转、凝视和镜像对话，都会形成“注意力痕迹”，逐步提高对象的显影程度。

```text
模糊体验
    ↓
碎片与光尘
    ↓
沉积
    ↓
对象轮廓
    ↓
用户命名
    ↓
对象进入关系网络
    ↓
形成可被看见的模式
```

### 8.3 对象主动靠近意识

探索不应完全由用户主动发起。

长期未被关注、但与近期经验相关的对象，可以通过非常克制的方式发出信号：远方微光、轻微脉冲、粒子方向变化或局部雾气波动。

禁止红点、通知轰炸和强提醒。

### 8.4 关系自然演化

对象之间的关系不应以显式连线为主，而应通过引力、距离、粒子流、同步呼吸、光尘迁移和空间扰动表现。

当用户长期围绕两个主题停留或表达时，系统可让两者之间逐渐出现轻微影响，但不替用户解释因果。

## 9. 时间与演化

《内在宇宙》采用心理时间，而不是效率时间。

对象可以在短期内改变亮度、呼吸频率或粒子活动；中期形成新的关系场、雾区变化或局部结构调整；长期才可能出现更深的整合，例如创伤边缘被照亮、旧对象稳定下来、未知区域逐渐显影。

产品不使用等级、治愈百分比、连续打卡或完成度。演化不是变得更正面，而是变得更完整、更可承受、更有关系感。

## 10. 镜像系统

镜像是用户意识与无意识之间的翻译界面。

它的语言必须遵守以下原则：

- 只描述可被观察到的现象，不做诊断。
- 优先提问与映照，不提供权威解释。
- 不制造依赖，不模仿咨询师关系。
- 不对用户的心理结构作确定性归因。
- 不承诺疗愈效果。

示例：

可用：
> 这里似乎一直在等待被注意。  
> 最近，这两个部分似乎开始彼此影响。  
> 当你靠近它时，最先出现的是什么？  
> 如果它有一个名字，你会怎样称呼它？

不可用：
> 你的问题来自童年。  
> 你患有……  
> 你应该立刻……  
> 恭喜你完成疗愈。

## 11. 首次体验

首次体验不应以注册、问卷、测评或生成报告开始。

建议的第一分钟：

1. 黑暗中出现极轻微的深空噪声与一句话：  
   “在你认识自己之前，你的内在已经存在。”
2. 一颗微弱、非球形的心理对象在远方被照亮。
3. 镜头缓慢拉开，显现少量对象与大面积未知空间。
4. 远处出现小而稳定的暖金 Self 种子。
5. 用户第一次触碰一个对象，进入短暂凝视。
6. 镜像只说一句：  
   “你注意到了它。它也一直在那里。”
7. 用户回到宏观宇宙，理解这不是报告，而是一个可被持续探索的世界。

## 12. 完整用户旅程

### 初期：建立观看关系

用户看到少量对象和大量未知。重点不是解释，而是理解“内在世界可以被看见”。

### 中期：发现重复结构

用户开始回访对象、命名对象、发现两个或多个对象之间存在稳定牵动。系统用低侵入方式呈现关系，不替用户定性。

### 长期：形成个人神话

用户逐渐拥有自己的对象、区域、名称、象征和关系结构。整个宇宙成为一种个人叙事：用户不再只描述“我最近不好”，而能说“这个部分最近离那个部分很近”。

长期目标不是宇宙被探索完，而是用户能够更成熟地与未知共处。

## 13. 视觉与交互总原则

产品应保持以下体验气质：

- 深蓝黑、暖金、暗紫与克制微青。
- 2.5D 深度优先于传统扁平信息布局。
- 大结构慢，小对象有生命，但无快速炫技动效。
- 留白、黑暗和沉默是必要内容。
- UI 是轻薄意识层，不是管理面板。
- 越接近对象，越少界面；越深入，越少解释。
- 所有交互都应有心理含义：拖拽是调整距离，缩放是改变意识距离，停留是允许显影，命名是主体化，回访是关系深化。

## 14. 安全边界

《内在宇宙》不替代心理咨询、医疗诊断或危机干预。

当用户表达自伤、自杀、他伤或急性危机风险时，镜像模式应停止常规探索性叙事，清晰建议用户联系当地紧急服务、危机热线或专业心理支持，并避免继续进行沉浸式创伤内容引导。

## 15. 产品成功标准

产品成功不以日活、连续打卡或“积极情绪比例”单独衡量。

更核心的成功信号包括：

- 用户能把它描述为“一个内在空间”，而不是“一个聊天 App”。
- 用户能辨认和回访自己的心理对象。
- 用户能感受到对象并非被系统强行定义。
- 用户愿意在未知与不确定中停留，而不是急于得到答案。
- 用户逐渐形成更丰富、更具象的自我叙事。

## 16. 一句话蓝图

> 《内在宇宙》不是帮助用户修复一个问题，而是让用户在一个持续显影的心理世界中，慢慢发现、命名并重新理解那个一直比意识更大的自己。
